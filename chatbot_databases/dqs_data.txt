Indicadores (KPI) y métricas para darle sentido a los datos
A partir de toda información extraída y tratada del ERP, se modela para poder generar una serie de Indicadores (KPI) y Métricas que darán respuesta a las preguntas más frecuentes que se pueda realizar cualquier persona de la organización:
Nuestro modelo semántico de ventas dispone de más de  100 métricas disponibles.
Solución segura
El modelo semántico además de contener toda la lógica de negocio formulada en las métricas permite también establecer quien puede acceder y a qué puede acceder. En este sentido se establecen 2 tipos de seguridad:
01
Seguridad Soft
Un primer nivel donde se limita el acceso a ciertos módulos del report, Seguridad Soft
02
Seguridad hard
Donde el usuario puede acceder a todos los módulos del informe pero no tiene acceso a toda la información, se realiza un filtro sobre aquellos datos a los que no puede tener acceso.
Solución personalizable y escalable
Escalable: al disponer de la información en el DWH, se puede complementar con información de varios ERPs o de orígenes diferentes, p.e, presupuestos, campañas de marketing, resultado de un mailing, redes sociales, … Con estas nuevas fuentes de datos, se puede ampliar el modelo semántico existente y generar nuevas métricas que permitirán ampliar la funcionalidad actual
Personalizable: la solución actual permite bien modificar los reports existentes, bien crear nuevos reports pero siempre manteniendo el mismo origen de los datos a través del modelo semántico
El modelo semántico está preparado para poder recoger información de diferentes fuentes de datos. Cada ERP podrá tener un conector especifico que permitirá realizar la extracción de información relevante de forma autónoma y automática.
En este proceso se preparará la información extraída del ERP para poder contestar las preguntas de que se realicen posteriormente desde negocio.

​​Toda esta información se procesará y guardará en un Datawarehouse (on premise, cloud o directamente en el modelo de datos) fuera del ERP, de esta manera se aíslan los entornos y cada uno de ellos puede seguir su operativa sin interferir entre ellos
A partir de estos procesos de extracción, transformación y carga, se define un Modelo Relacional robusto que permitirá establecer la lógica y las reglas de negocio necesarias para explorar la información
